## 🚀 Why Magna Docs?

Most documentation platforms require maintaining a completely separate website.

Whether you're using **GitBook**, **VitePress**, **Docusaurus**, **MkDocs**, or **Docsify**, you're often managing:

- a separate repository
- Node.js dependencies
- build pipelines
- deployment workflows
- synchronization between your application and documentation

Magna Docs takes a different approach.

It lives inside your **Laravel application** and becomes part of your CMS.

Every page is stored in your database, rendered using Blade, cached automatically, and available immediately after publishing.

There is nothing extra to build, deploy, or maintain.

# 🚀 Installation

Installing Magna Docs takes only a few minutes.

Since Magna Docs is a native Magna CMS plugin, there are no Node.js dependencies, frontend build tools, or additional services to configure.

## Requirements

Before installing Magna Docs, ensure your environment meets the following requirements.

| Requirement | Version |
|-------------|---------|
| PHP | 8.3 or higher |
| Laravel | 11.x |
| Magna CMS | Latest Version |
| Filament | Version 3 |
| Composer | Latest Version |

---

## Step 1 — Place the Plugin

Copy or clone the plugin into your local plugins directory.

```
plugins-dev/
└── magna/
    └── docs/
```

Your project structure should now look similar to:

```
your-project/
│
├── app/
├── bootstrap/
├── config/
├── plugins-dev/
│   └── magna/
│       └── docs/
├── public/
├── routes/
└── vendor/
```

---

## Step 2 — Register the Repository

Open your project's `composer.json`.

Add the plugin repository under the `repositories` section.

```json
{
    "repositories": [
        {
            "type": "path",
            "url": "plugins-dev/magna/docs"
        }
    ]
}
```

---

## Step 3 — Install the Package

Run:

```bash
composer require magna/docs:@dev
```

Composer will automatically install all required dependencies.

---

## Step 4 — Run Database Migrations

```bash
php artisan migrate
```

This creates the required database tables for:

- Documentation pages
- Collections
- Plugin settings
- Feedback records (if enabled)

---

## Step 5 — Enable the Plugin

Login to your Magna CMS admin panel.

Navigate to:

```
Admin
    → Plugins
        → Magna Docs
```

Click **Enable Plugin**.

Once enabled, a new **Magna Docs** section will appear in the admin sidebar.

---

# 🎉 You're Ready!

Visit:

```
Admin
    → Magna Docs
```

Create your first documentation page and publish it instantly.

No build process required.

## Authentication

By default, all published documentation endpoints are publicly accessible.

Protected endpoints should be secured using your application's authentication and authorization mechanisms.

---

## API Endpoints

### Get Documentation Tree

Returns the complete documentation hierarchy.

```http
GET /api/v1/docs/tree
```

Example Response

```json
[
    {
        "title": "Getting Started",
        "slug": "getting-started",
        "children": [
            {
                "title": "Installation",
                "slug": "installation"
            },
            {
                "title": "Configuration",
                "slug": "configuration"
            }
        ]
    }
]
```

---

### List All Published Pages

```http
GET /api/v1/docs/pages
```

Returns a flat list of every published page.

Useful for:

- Search indexing
- Static exports
- Mobile apps

---

### Get a Single Page

```http
GET /api/v1/docs/pages/{slug}
```

Example

```http
GET /api/v1/docs/pages/installation
```

Returns:

- title
- slug
- content
- metadata
- featured image
- breadcrumb
- reading time
- last updated date
